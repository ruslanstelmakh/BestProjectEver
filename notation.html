Литеральные объекты

Литеральный объект – это объект, определенный в виде списка атрибутов, перечисленных через запятую и заключенных в фигурные скобки. Значе- ние атрибута отделяется от имени двоеточием, а не знаком равенства. Литеральные объекты могут также содержать массивы, представленные в виде списка элементов, перечисленных через запятую и заключенных в квадратные скобки. Для определения методов в качестве значения од- ного из атрибутов задается функция.

var spa = { title: "Single Page Web Applications", //атрибут authors: [ "Mike Mikowski", "Josh Powell" ], //массив buy_now: function () { //функция console.log( "Book is purchased" ); } }

Глобальные переменные и объект window

То, что мы привыкли называть глобальными переменными, – на самом деле свойства объекта верхнего уровня, представляющего среду выполнения. В браузере таковым является объект window ; в Node.js – объект global, и область видимости переменных работает по-разному. Объект window содержит много свойств, которые сами по себе содержат объекты, методы (onload , onresize , alert , close ...), элементы DOM (document , frames ...), а также другие переменные. Доступ ко всем этим свойствам осуществляется с помощью синтаксиса window.property.

window.onload = function(){ window.alert('window loaded'); }

Объект верхнего уровня в Node.js называется global. Поскольку Node. js – веб-сервер, а не браузер, множество доступных функций и свойств существенно отличается. Когда интерпретатор JavaScript, работающий в браузере, проверяет существование глобальной переменной, он просматривает объект window.

var regular_joe = 'Глобальная переменная'; console.log( regular_joe ); // 'Глобальная переменная' console.log( window.regular_joe ); // 'Глобальная переменная' console.log( regular_joe === window.regular_joe ); // true

Использование Object.create для создания объектов.

В качестве альтернативы оператору new был разработан метод Object.create. Метод Object.create принимает прототип в качестве аргумента и возвращает объект; таким образом, можно определить объект-прототип с общими атрибутами и методами и использовать его для создания других объектов с такими же свойствами.

var proto = { sentence : 4, probation : 2 }; var firstPrisoner = Object.create( proto ); firstPrisoner.name = 'Joe'; firstPrisoner.id = '12A'; var secondPrisoner = Object.create( proto ); secondPrisoner.name = 'Sam'; secondPrisoner.id = '2BC'; console.log(firstPrisoner.sentence); console.log(secondPrisoner.probation);

Существует и другой, тоже широко распространенный, подход к использованию Object.create – написать с его помощью фабричную функцию, которая создает и возвращает объект.

var proto = { sentence : 4, probation : 2 }; var makePrisoner = function( name, id ) { // makePrisoner – фабричная функция, она создает объекты prisoner var prisoner = Object.create( proto ); // Само создание объекта производится так же, prisoner.name = name; как в листинге выше, prisoner.id = id; просто оно обернуто фабричной функцией. return prisoner; }; // Теперь для создания новых объектов достаточно вызвать функцию makePrisoner, передав ей атрибуты свойств. var firstPrisoner = makePrisoner( 'Joe', '12A' ); var secondPrisoner = makePrisoner( 'Sam', '2BC' );

Если свойство отсутствует у самого объекта, то проверяется его наличие в прототипе
